راهنمای تست فیلترهای آبشاری - فرم سفارش V2

⚠️ پیش‌نیازها

  • وردپرس نصب و فعال باشد
  • افزونه Tabesh نصب و فعال باشد
  • Pricing Engine V2 فعال باشد (تنظیمات → قیمت‌گذاری محصول)
  • حداقل یک قطع کتاب با محدودیت‌ها پیکربندی شده باشد
  • فرم با shortcode [tabesh_order_form_v2] در یک صفحه نمایش داده شود

تست 1️⃣: بارگذاری اولیه فرم

مرحله 1: باز کردن صفحه فرم

اقدام: صفحه‌ای که shortcode فرم را دارد باز کنید

انتظار:
  • فرم به درستی نمایش داده شود
  • قطع‌های کتاب در dropdown موجود باشند
  • فیلدهای بعدی مخفی باشند
نتیجه واقعی: (توسط تست‌کننده پر شود)
  • ☐ قطع‌ها نمایش داده می‌شوند
  • ☐ فیلدهای بعدی مخفی هستند

مرحله 2: انتخاب قطع کتاب

اقدام: یک قطع (مثلاً "A5") را انتخاب کنید

انتظار:
  • Loading indicator نمایش داده شود
  • درخواست AJAX به /wp-json/tabesh/v1/get-allowed-options ارسال شود
  • فیلد "نوع کاغذ" نمایش داده شود
  • لیست کاغذهای مجاز بارگذاری شود
  • Console log: Book size selected: A5
  • Console log: Allowed options response: {success: true, ...}
نتیجه واقعی:
  • ☐ Loading نمایش داده شد
  • ☐ AJAX request در Network tab دیده می‌شود
  • ☐ لیست کاغذها بارگذاری شد

تست 2️⃣: فیلتر آبشاری - تغییر نوع کاغذ

مرحله 1: انتخاب نوع کاغذ

اقدام: یک نوع کاغذ (مثلاً "گلاسه") را انتخاب کنید

انتظار:
  • درخواست AJAX جدید به /get-allowed-options ارسال شود
  • این بار با پارامتر current_selection: {paper_type: "گلاسه"}
  • لیست صحافی‌ها بروزرسانی شود (فقط صحافی‌های مجاز برای گلاسه)
  • Console log: Paper type selected: گلاسه
  • Console log: Refreshing allowed options with selection: {paper_type: "گلاسه"}
  • Console log: Refreshed options response: ...
نتیجه واقعی:
  • ☐ AJAX request با paper_type ارسال شد
  • ☐ لیست صحافی‌ها بروزرسانی شد
  • ☐ Console logs صحیح هستند

مرحله 2: تغییر نوع کاغذ

اقدام: نوع کاغذ را به "تحریر" تغییر دهید

انتظار:
  • درخواست AJAX جدید با paper_type: "تحریر"
  • لیست صحافی‌ها دوباره بروزرسانی شود
  • اگر صحافی انتخاب شده بود و هنوز معتبر است → حفظ شود
  • اگر صحافی انتخاب شده دیگر معتبر نیست → پاک شود
نتیجه واقعی:
  • ☐ AJAX با paper_type جدید ارسال شد
  • ☐ لیست صحافی‌ها به درستی بروزرسانی شد
  • ☐ انتخاب قبلی به درستی مدیریت شد

تست 3️⃣: Reset خودکار فیلدهای بعدی

سناریو کامل

اقدام:

  1. تمام فیلدها را پر کنید (از قطع تا نوع صحافی)
  2. قطع کتاب را تغییر دهید
انتظار:
  • تمام فیلدهای بعد از قطع پاک شوند
  • فقط فیلدهای قطع و عنوان کتاب باقی بمانند
  • کاربر مجبور شود از ابتدا فرم را پر کند
نتیجه واقعی:
  • ☐ فیلدها به درستی reset شدند
  • ☐ فرم به حالت اولیه برگشت

تست 4️⃣: جریان کامل سفارش

سناریو کامل

اقدام:

  1. عنوان کتاب: "تست"
  2. قطع: A5
  3. نوع کاغذ: تحریر
  4. گرماژ کاغذ: 70
  5. نوع چاپ: سیاه و سفید
  6. تعداد صفحات: 100
  7. تیراژ: 100
  8. نوع صحافی: شومیز
  9. گرماژ جلد: 200
  10. محاسبه قیمت
  11. ثبت سفارش
انتظار:
  • قیمت به درستی محاسبه شود
  • سفارش با موفقیت ثبت شود
  • هیچ خطای "ترکیب نامعتبر" نمایش داده نشود
  • پیام موفقیت نمایش داده شود
  • کاربر به صفحه سفارشات هدایت شود
نتیجه واقعی:
  • ☐ قیمت محاسبه شد
  • ☐ سفارش ثبت شد
  • ☐ خطایی رخ نداد
  • ☐ redirect انجام شد

تست 5️⃣: Network و Console Debugging

بررسی درخواست‌های AJAX

نحوه بررسی: Developer Tools → Network tab → فیلتر XHR

درخواست‌های مورد انتظار:
  • ✅ POST /wp-json/tabesh/v1/get-allowed-options (انتخاب قطع)
  • ✅ POST /wp-json/tabesh/v1/get-allowed-options (انتخاب کاغذ)
  • ✅ POST /wp-json/tabesh/v1/get-allowed-options (انتخاب گرماژ)
  • ✅ POST /wp-json/tabesh/v1/get-allowed-options (انتخاب صحافی)
  • ✅ POST /wp-json/tabesh/v1/calculate-price (محاسبه قیمت)
  • ✅ POST /wp-json/tabesh/v1/submit-order (ثبت سفارش)

همه درخواست‌ها باید Status Code 200 یا 201 داشته باشند

نتیجه واقعی: (تعداد و وضعیت درخواست‌ها را بنویسید)

بررسی Console Logs

نحوه بررسی: Developer Tools → Console tab

پیام‌های مورد انتظار:
  • Initializing Tabesh Order Form V2...
  • Book size selected: [قطع انتخابی]
  • Paper type selected: [نوع کاغذ]
  • Refreshing allowed options with selection: {...}
  • Refreshed options response: {success: true, ...}
  • هیچ error قرمز رنگی نباید وجود داشته باشد
نتیجه واقعی: (خطاها یا پیام‌های غیرعادی را بنویسید)
